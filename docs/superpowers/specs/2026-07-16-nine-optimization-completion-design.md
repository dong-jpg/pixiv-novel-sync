# 九项历史优化收尾设计

日期：2026-07-16
状态：已确认方向，等待规格复核

## 1. 背景与目标

本次工作用于收尾九项历史优化。审计确认第 1、7 项已经完成；第 2、3、4、6、8、9 项仍有功能或交互缺口；第 5 项功能可用，但页面结构需要针对性重构。

目标不是重写现有系统，而是在保持现有 Flask、Vue 3、SQLite 和 Library OS 设计语言的前提下完成以下闭环：

1. 让 AI 清洗后的关键词真正进入偏好画像、搜索计划和 Pixiv 搜索。
2. 让同步任务与 AI 创作任务在统一任务日志页获得一致、可用的筛选和详情体验。
3. 让首页、AI 项目总览和风格控制区域布局整齐，并消除“设置了但没有保存”的误导。
4. 让 AI 创作小说具备可持久化的封面，并在小说库和阅读页复用同一展示规则。
5. 让风格控制覆盖创作链的关键写作阶段，而不仅是单次章节续写。
6. 将创作向导与自动写作从同一份超大模板中拆开，避免无关数据和代码互相牵连。
7. 更新过期文档，补足缺失测试，并只清理确定属于缓存的文件。

## 2. 范围与非目标

### 2.1 本次范围

- 偏好分析、关键词清洗、画像重建和推荐搜索计划。
- 同步任务及 AI 创作任务的统一日志列表、筛选、详情和保留周期。
- 首页“最近活动”“定时任务”“最近推书结果”的布局。
- AI 创作项目封面上传、读取、替换和删除。
- AI 项目级风格控制的保存、展示和生成链注入。
- 自动写作页、创作向导页及蒸馏档案页的模板边界。
- 与上述功能直接相关的文档和测试。

### 2.2 非目标

- 不迁移历史 `recommendation_runs` 或旧版本 AI 日志到新表。
- 不把任务运行中的内存日志改造成实时持久化流；统一页仍以任务状态和已持久化结果为准。
- 不重写前端框架，不引入 SPA 路由或新的构建工具。
- 不引入在线图片生成服务；封面由用户上传，未上传时继续使用稳定渐变占位。
- 不删除 `.env`、`config/config.yaml`、`db.sqlite`、`data/`、`memory/` 或 `.claude/` 中的个人文件。
- 不对与九项优化无关的 AI 服务、同步引擎或数据库模块做广泛重构。

## 3. 总体实施顺序

### 阶段 A：功能断链修复

先修复关键词清洗和统一任务日志。这两部分影响真实搜索结果和任务排障，优先级最高。

### 阶段 B：AI 作品与风格控制

增加本地封面能力，补齐风格保存入口和关键生成链注入，同时重排项目总览卡片。

### 阶段 C：页面边界与首页布局

拆分自动写作和创作向导模板，移除无关 API 加载；完成首页等高布局和响应式检查。

### 阶段 D：文档、清理与验证

更新前端文档和 README，清理缓存，执行定向测试、完整测试和 Git 状态复核。

各阶段使用独立提交。上一阶段测试通过后才进入下一阶段。

## 4. 偏好关键词清洗闭环

### 4.1 单一分析流程

手动分析和定时分析必须复用同一个偏好分析服务。该服务接收批次数、批大小、重置标记、日志报告器和取消回调，不再在 `jobs/tasks.py` 与 `web/managers.py` 中维护两套画像保存逻辑。

- 手动分析默认最多处理 10 批。
- 定时分析每次只处理 1 批。
- 两种入口均写入 `task_logs`。
- 没有新小说时记录“暂无新的小说需要分析”，不调用 AI。

### 4.2 清洗与画像重建顺序

规范顺序为：

1. 从累加器重建原始 `stats`。
2. 取 `top_keywords` 和 `top_tags` 调用 `keyword_clean` Agent。
3. 清洗成功时写入 `stats.refined_keywords` 和 `stats.refined_keywords_dropped_sample`。
4. 使用更新后的 `stats` 重新生成 `profile`。
5. 保存完整画像。

画像构建器优先使用 `refined_keywords`，为空或 AI 不可用时回退到 `top_keywords`。`positive_preferences.keywords`、`summary`、`precise_queries` 和 `experimental_queries` 均使用同一组最终关键词，避免展示结果和搜索策略不一致。

### 4.3 Agent 配置

设置页的 Agent 类型补充 `keyword_clean`。内置“关键词清洗师”继续作为首选；没有专用 Agent 时允许回退到 `general`；调用失败时偏好分析仍成功，只记录警告并使用原始关键词。

### 4.4 验收条件

- 手动和定时偏好分析都会触发相同清洗流程。
- 清洗结果会进入保存后的画像和搜索计划。
- `RecommendationService.build_search_plan()` 产生的查询包含清洗词，不包含已被清洗丢弃的口语噪声词。
- 无 Agent、无关键词、非法 JSON 或 Provider 失败时保持旧行为，不中断偏好分析。

## 5. 统一任务日志闭环

### 5.1 数据源保持分离

继续保留 `task_logs` 与 `ai_jobs` 两张表，不做数据搬迁。`GET /api/dashboard/logs` 根据 `category=sync|ai` 分流，并把两种数据规范化为统一列表结构。

AI 列表项至少返回：

- `category`
- `job_id`
- `task_type`
- `task_name`
- `status`
- `started_at`
- `finished_at`
- `duration_seconds`
- `error_message`

AI 列表不再伪造空的数字 `id`。页面展示任务标识时，同步任务显示 `id`，AI 任务显示 `job_id`。

### 5.2 AI 详情

点击 AI 行后调用现有 `GET /api/dashboard/ai/jobs/<job_id>`，不再直接把残缺的列表行当详情使用。详情弹窗按存在性展示：

- 输入参数 `input`
- 文本输出 `output_text`
- 结构化输出 `output`
- 错误信息
- 开始、结束和耗时

大段文本使用有高度上限的滚动区，并提供复制按钮。前端不得使用 `v-html` 渲染 AI 输出或错误文本。

### 5.3 筛选与类型映射

AI 分类补充状态筛选，支持全部、运行中、成功、失败和已取消。后端把 `status` 传入 AI 查询。

任务类型映射至少覆盖当前真实类型：

- `chapter_continue`
- `chapter_pipeline`
- `longform_plan`
- `longform_plan_details`
- `continue`
- `rewrite`
- `audit`
- `plan`
- `distill_style`
- `distill_novel`
- `extract_summary`
- `summarize`
- `chat`
- `update_state`
- `state_update`
- `resolve_foreshadow`
- `foreshadow_resolve`
- `polish`
- `polish_dialogue`
- `polish_psychology`
- `keyword_clean`

后端、日志页和 AI 页面共享一份规范映射，至少通过集中常量或测试保证一致，不继续维护三份无约束字典。

### 5.4 三天保留规则

- `TasksMixin.cleanup_old_task_logs()` 默认 3 天。
- AI 存储清理默认 3 天。
- AI 服务层 `cleanup_jobs()` 默认改为 3 天。
- 日志页面只提供 1 天和 3 天查询，不再提供无法返回数据的 7 天选项。
- README 和前端文档统一写“默认保留 3 天”。

### 5.5 验收条件

- 两个分类均可按类型、时间和适用状态筛选。
- AI 任务详情能看到真实 `job_id`、输入、输出和错误。
- 已取消状态显示中文。
- 自动清理、手工清理和服务默认值均为 3 天。

## 6. AI 创作小说封面

### 6.1 数据模型

为 `ai_writing_projects` 增加可空的 `cover_path` 字段，保存相对于公共存储根目录的路径，不保存客户端提供的绝对路径。数据库迁移必须可重复执行，并兼容现有项目。

API 返回项目时增加计算后的 `cover_url`。未设置封面时 `cover_url` 为空，前端继续使用现有书名渐变占位。

### 6.2 上传与读取

新增经过现有 Dashboard 鉴权和 CSRF 保护的接口：

- `POST /api/dashboard/ai/projects/<project_id>/cover`：上传或替换封面。
- `GET /api/dashboard/ai/projects/<project_id>/cover`：读取封面。
- `DELETE /api/dashboard/ai/projects/<project_id>/cover`：删除封面并恢复渐变占位。

上传限制：

- 仅接受 JPEG、PNG、WebP。
- 最大 10 MiB。
- 同时检查扩展名、MIME 和文件头。
- 文件写入 `public_dir/ai_projects/<project_id>/cover.<ext>`。
- 使用原子写入，替换成功后删除旧扩展名文件。
- 所有解析后的路径必须确认位于 `public_dir` 内。

删除 AI 项目时同时删除其封面目录；数据库删除成功但文件已不存在时仍视为成功。

### 6.3 页面复用

小说库 AI 创作 Tab、AI 阅读页和自动写作项目总览使用相同规则：优先显示 `cover_url`，加载失败或不存在时显示稳定渐变与书名首字符。

项目总览提供“上传封面”“替换封面”“删除封面”操作，不引入独立封面管理页面。

## 7. 风格控制与项目总览布局

### 7.1 保存语义

风格控制卡片内增加明确的“保存风格设定”按钮和保存结果提示。蒸馏档案卡片只保存档案绑定，不再让用户依赖另一张卡片的按钮保存风格。

保存设置时必须合并当前 `settings`，不能覆盖 `longform_plan` 等其他项目设置。启动章节续写或 Pipeline 前仍执行一次安全保存，避免用户调整后直接生成导致设置丢失。

### 7.2 布局

项目总览把四张卡片改为同一个两列网格中的直接子项：

1. 项目信息。
2. 项目概况。
3. 套用蒸馏档案。
4. 风格控制。

同一行卡片使用 `h-full`，在桌面端上下边界对齐；移动端按上述顺序单列展示。风格卡片继续沿用白底、圆角、浅边框、阴影和品牌色按钮，不引入独立视觉体系。

### 7.3 注入范围

项目级风格控制应用于会产出正文或直接影响正文风格的路径：

- 长篇规划和详细梗概：作为全书风格约束传入规划 Prompt。
- 章节续写及 Pipeline 的续写步骤。
- 项目章节的对话润色和心理描写润色。

审计、状态提取、摘要、检索索引等非正文生成任务不注入风格控制，避免污染结构化输出。

滑块值为 50 时保持中性，不产生对应指令；无设置的历史项目保持当前行为。

## 8. 自动写作与创作向导模板拆分

### 8.1 页面边界

- `/dashboard/ai` 渲染自动写作专用模板，只包含项目、规划、章节、伏笔、状态和检索功能。
- `/dashboard/wizard` 渲染创作向导专用模板，只包含创作向导和蒸馏档案。

两页继续共享基础布局、通用弹窗和必要的小组件，但不再依赖 `pageMode` 隐藏整套无关页面。

### 8.2 数据加载

自动写作页不请求创作向导会话；创作向导页不请求章节产出面板和项目工作区的无关数据。共享的 Agent、风格档案、小说档案加载可以保留，但必须由实际页面需要触发。

创作向导导入项目后跳转 `/dashboard/ai?project_id=<id>`，自动打开目标项目，维持现有工作流。

### 8.3 重构边界

只拆分模板和前端状态边界，不改变现有后端服务协议。若存在两页都使用的组件，抽为 Jinja include 或小型共享脚本；不创建新的前端框架层。

## 9. 首页布局与推荐结果

“最近推书结果”继续使用 `GET /api/dashboard/recommendations/items?limit=6`，保持独立全宽模块。

“最近活动”和“定时任务”在桌面端位于同一网格行，外层卡片和内部主体均使用可拉伸布局，保证边框底部对齐；移动端自然堆叠，不使用固定高度。

推荐卡片必须覆盖加载、空结果、接口失败、单篇和系列跳转。接口失败时显示可见的轻量错误与重试入口，不再只写入控制台后伪装成正常空态。

## 11. 错误处理与兼容性

- AI 清洗失败只降级，不让偏好分析失败。
- AI 日志详情不存在时返回 404，页面保留列表行并显示“详情已清理或不存在”。
- 封面上传验证失败返回 400；磁盘写入失败不更新数据库；数据库更新失败时删除新写入的临时文件。
- 旧项目没有 `cover_path` 或 `style_control` 时正常显示渐变封面和中性风格。
- 模板拆分后保留现有深链接 `/dashboard/ai?project_id=<id>` 与 `/dashboard/wizard?tab=distill`。
- 所有新增修改接口继续使用现有 CSRF 防护。

## 12. 测试设计

### 12.1 偏好与推荐

- 清洗成功后保存画像使用 `refined_keywords`。
- 搜索计划使用清洗词并排除丢弃词。
- 定时与手动分析调用同一服务，定时限制为一批。
- 无 Agent 和 Provider 失败时降级。

### 12.2 任务日志

- AI 投影包含规范字段和完整类型映射。
- `category=ai` 支持类型、状态和天数过滤。
- AI 详情通过 `job_id` 返回输入、文本输出和结构化输出。
- 三天清理默认值在存储、服务和路由保持一致。
- 页面不再包含 7 天选项。

### 12.3 封面

- 数据库迁移兼容已有表。
- 合法图片可上传、读取、替换和删除。
- 非图片、伪造文件头、超限文件和路径穿越被拒绝。
- 删除项目清理封面。
- 小说库和阅读页都使用 `cover_url` 并保留渐变回退。

### 12.4 风格与模板

- 项目设置合并不覆盖其他字段。
- 风格控制进入规划、续写和两种润色 Prompt。
- 中性或缺失设置不产生风格指令。
- `/dashboard/ai` 不包含创作向导主界面，`/dashboard/wizard` 不包含自动写作工作区。
- 两个页面只加载各自需要的数据。
- 点击章节仍进入“本章节”Tab，章节产出 Tab 保持可用。

### 12.5 验证命令

先运行新增和受影响的定向测试，再运行：

```powershell
python -m pytest -q
```

完成标准为全部测试通过；既有环境相关跳过项数量不得无故增加。
